# 拍摄上传命名对齐与自动化处理部署指南（对齐 EmbodMocap / Spectacular Rec）

目标：让本项目（`spatial_data_recorder`）产出的 ZIP 上传到服务器后，**无需人工改名/搬运**，即可被 `EmbodMocap` 的 `auto_spectacular_rec_service.py` 自动识别并：

- **纯场景（scene-only）**：自动跑 **Step0-2**，产出 `transforms.json`、`mesh_raw.ply / mesh_simplified.ply`，并用 Viser 预览
- **带人（human）**：当同一 `scene + seq` 的 **A/B 两个视角**齐全后，自动跑 **Step0-15** 主流程，并用 Viser 展示

本指南把“要用到的命令”集中到一个文档里，便于复制执行。

---

## 1. ZIP 文件名规范（最重要）

服务器端 `EmbodMocap` 的自动化服务按 **文件名 token** 判断要走的流程（scene-only / human）。因此后端必须**保留客户端上传时的原始文件名**，并把 ZIP 放到 `_incoming/` 目录下。

统一格式：

```text
recording_<任意原始部分>__scene=<SCENE>__type=<scene|human>[__seq=seq0][__cam=A|B].zip
```

字段含义：

- `scene=<SCENE>`：场景目录名，最终会落盘到 `DATA_ROOT/<SCENE>/`
- `type=scene`：纯场景（只建图/mesh），自动跑 Step0-2
- `type=human`：带人拍摄（需要双视角），自动跑 Step0-15
- `seq=seq0/seq1/...`：human 必填，必须是 `seq` + 数字
- `cam=A|B`：human 必填（除非你使用 `stereo=1` bundle；本项目当前按 A/B 双文件方式对齐）
  - `cam=A` → `raw1`
  - `cam=B` → `raw2`

### 1.1 本项目字段如何映射到文件名

本项目会把会话目录里的 `upload_context.json`（`UploadSessionContext`）映射到 ZIP 文件名：

- `sceneName` → `scene=...`
- `captureType`
  - `sceneOnly` → `type=scene`
  - `humanInScene` → `type=human`
- `seqName` → `seq=...`（human 才会写入文件名）
- `cam` → `cam=A|B`（human 必填）

### 1.2 命名示例（建议直接照抄改时间戳）

纯场景（上传 1 个 zip）：

```text
recording_2026-04-16_10-20-30__scene=scene_20260416_102030_livingroom1__type=scene.zip
```

带人（同一 take：`seq0`，上传 2 个 zip：A/B 各一个）：

```text
recording_2026-04-16_10-45-00__scene=scene_20260416_102030_livingroom1__type=human__seq=seq0__cam=A.zip
recording_2026-04-16_10-45-00__scene=scene_20260416_102030_livingroom1__type=human__seq=seq0__cam=B.zip
```

> 注意：如果 human 模式只上传了 A 或只上传了 B，服务器端不会触发完整 Step0-15（因为 Step4 需要双视角输入）。

## 2. 服务器目录约定（DATA_ROOT）

以 `datasets/my_capture` 为例：

```bash
# 在 EmbodMocap 仓库下执行（示例）
cd embod_mocap

export DATA_ROOT=../datasets/my_capture
mkdir -p "$DATA_ROOT/_incoming"
```

约定：

- 客户端上传的 ZIP 先落到：`$DATA_ROOT/_incoming/`
- 自动化服务会把它们搬运/解包到：
  - scene-only：`$DATA_ROOT/<SCENE>/`
  - human：`$DATA_ROOT/<SCENE>/<SEQ>/recording_*.zip`（后续 Step4 自动整理到 `raw1/raw2`）

## 3. 启动自动化处理服务（核心：auto_spectacular_rec_service）

在服务器上运行（建议常驻后台，例如 `tmux` / `systemd`）：

```bash
cd embod_mocap

conda run -n embodmocap python tools/auto_spectacular_rec_service.py \
  --data_root ../datasets/my_capture \
  --config config_fast.yaml \
  --conda conda \
  --env_main embodmocap \
  --env_sai embodmocap_sai150 \
  --mode skip
```

只想“导入并整理目录，不自动跑 Step”时加：

```bash
--no_auto_run
```

> 说明：该服务会轮询扫描 `$DATA_ROOT/_incoming/`，自动识别 `type=scene|human` 并触发对应步骤。Step1/Step5 通常会在 `embodmocap_sai150` 环境执行，其余步骤在 `embodmocap` 主环境执行。

## 4. （可选）部署一个最小上传接收端（HTTP → _incoming）

如果你希望让手机 App 直接上传，而不是用 `scp`/网盘中转，后端只需要做到一件事：

- **把 multipart 里的 `file` 按原始文件名**保存到：`$DATA_ROOT/_incoming/<filename>`

### 4.1 用 scp/rsync 手工投递 ZIP（最省事）

```bash
scp recording_*.zip <user>@<server>:/path/to/datasets/my_capture/_incoming/
```

### 4.2 用 FastAPI 起一个简易上传服务（示例命令）

```bash
mkdir -p ~/sdr_upload_service
cd ~/sdr_upload_service

python -m venv .venv
source .venv/bin/activate
pip install fastapi uvicorn python-multipart

# DATA_ROOT 指向 EmbodMocap 的 datasets/my_capture
export DATA_ROOT=/path/to/datasets/my_capture
export UPLOAD_AUTH_TOKEN=slam-self-use-token

# 生成一个最小上传接收端（保存为 upload_receiver.py）
cat > upload_receiver.py <<'PY'
import os
import shutil
from pathlib import Path
from typing import Optional

from fastapi import FastAPI, File, Form, Header, HTTPException, UploadFile

app = FastAPI()


def _require_env(key: str) -> str:
    v = (os.environ.get(key) or "").strip()
    if not v:
        raise RuntimeError(f"Missing env: {key}")
    return v


def _unique_path(dst: Path) -> Path:
    if not dst.exists():
        return dst
    stem, suffix = dst.stem, dst.suffix
    for i in range(1, 1000):
        cand = dst.with_name(f"{stem}({i}){suffix}")
        if not cand.exists():
            return cand
    raise RuntimeError(f"Too many duplicates: {dst.name}")


@app.post("/upload")
async def upload(
    file: UploadFile = File(...),
    authorization: Optional[str] = Header(default=None),
    sessionName: Optional[str] = Form(default=None),
    sessionPath: Optional[str] = Form(default=None),
    captureName: Optional[str] = Form(default=None),
    sceneName: Optional[str] = Form(default=None),
    seqName: Optional[str] = Form(default=None),
    captureType: Optional[str] = Form(default=None),
    pairGroupId: Optional[str] = Form(default=None),
    audioTrackPresent: Optional[str] = Form(default=None),
):
    data_root = Path(_require_env("DATA_ROOT")).expanduser().resolve()
    incoming = data_root / "_incoming"
    incoming.mkdir(parents=True, exist_ok=True)

    expected_token = (os.environ.get("UPLOAD_AUTH_TOKEN") or "").strip()
    if expected_token:
        if not authorization or not authorization.startswith("Bearer "):
            raise HTTPException(status_code=401, detail="Missing Authorization: Bearer <token>")
        token = authorization.split(" ", 1)[1].strip()
        if token != expected_token:
            raise HTTPException(status_code=403, detail="Invalid token")

    filename = Path(file.filename or "upload.zip").name
    dst = _unique_path(incoming / filename)

    with dst.open("wb") as f:
        shutil.copyfileobj(file.file, f)

    return {
        "ok": True,
        "saved_as": dst.name,
        "bytes": dst.stat().st_size,
        "sessionName": sessionName,
        "sceneName": sceneName,
        "seqName": seqName,
        "captureType": captureType,
        "pairGroupId": pairGroupId,
        "audioTrackPresent": audioTrackPresent,
        "captureName": captureName,
        "sessionPath": sessionPath,
    }
PY

uvicorn upload_receiver:app --host 0.0.0.0 --port 9000
```

客户端（本项目）侧 `.env` 配置建议：

```text
UPLOAD_BASE_URL=http://<server>:9000
UPLOAD_PATH=/upload
UPLOAD_AUTH_TOKEN=slam-self-use-token
```

> `upload_receiver.py` 的核心逻辑：接收 `file` → 保存到 `$DATA_ROOT/_incoming/`，并返回 JSON；并用 `Authorization: Bearer ...` 做最小鉴权（与本项目 `.env` 对齐）。

## 5. Viser 预览命令（场景 / 全流程）

### 5.1 仅预览场景 mesh（推荐用于 scene-only）

```bash
cd embod_mocap
conda run -n embodmocap python tools/preview_scene_meshes_viser.py \
  --data_root ../datasets/my_capture \
  --port 8080
```

端口冲突时改端口（例如 8081）：

```bash
--port 8081
```

### 5.2 预览全量结果（human 主流程）

如果你有用于可视化的 xlsx（例如自动流程产出的 `seq_info_*.xlsx`），可用：

```bash
cd embod_mocap
conda run -n embodmocap python tools/visualize_viser.py \
  --xlsx ../datasets/release_demo.xlsx \
  --data_root ../datasets/my_capture \
  --stride 2 \
  --scene_mesh simple \
  --mesh_level 1 \
  --port 8080
```

### 5.3 SSH 端口转发（本地浏览器看服务器 Viser）

```bash
ssh -p 22 -L 18080:127.0.0.1:8080 <user>@<server>
```

然后本地浏览器打开：

```text
http://127.0.0.1:18080
```

## 6. 常见失败点（对齐时优先查）

1) 上传后被丢进 `_incoming/_bad/`

- 重点检查文件名 token：
  - 是否包含 `__scene=...__type=...`
  - human 是否包含 `__seq=seq0` 与 `__cam=A|B`

2) human 只跑了部分步骤/不触发 Step0-15

- 同一 `scene + seq` 是否已经有 **A/B 两个 zip** 都上传到位

3) Viser 打不开 / 8080 被占用

- 改 `--port`，或修改 SSH 本地映射端口（例如 `18081:...:8080`）
