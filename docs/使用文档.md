# Spatial Data Recorder 使用文档

## 1. 文档说明

本文档面向当前**不包含扫码建组功能**的版本，重点说明如何完成以下操作：

- 开始录制前设置 `scene` / `seq`
- 手动输入双目分组号
- 开始与停止录制
- 在本地按 `scene/seq` 浏览录制结果
- 手动选择录制目录并上传

如果后续版本增加“扫码加入同步组”功能，应以新的使用文档为准。

## 2. 软件用途

本软件用于在 iPhone 上采集空间数据，会在每次录制时保存如下内容：

- 视频文件
- 传感器数据
- 标定信息
- 录制元数据
- 上传业务元数据

每次录制都会在本地生成一个独立会话目录，并可在后续选择上传到服务器。

## 3. 使用前准备

开始使用前，请确认：

- 使用的是 iPhone 真机
- App 已安装完成
- 已授予相机权限
- 如果需要录音，建议授予麦克风权限
- 已知本次录制应归属的 `scene` 和 `seq`

如果本次是双目同步采集，建议在开始前先和另一位拍摄者约定好：

- 使用同一个 `sceneName`
- 使用同一个 `seqName`
- 使用同一个 `pairGroupId`

注意：

- 当前版本**没有扫码功能**
- 如果要把两个不同设备拍摄的视频视为同一组，必须手动输入相同的分组号
- 如果希望它们上传到服务器同一个目录，除了分组号相同，还必须保证 `sceneName` 和 `seqName` 也相同

## 4. 主流程概览

一次完整使用流程如下：

1. 打开 App
2. 点击录制按钮
3. 在“录制前设置”面板中填写本次录制信息
4. 确认后开始录制
5. 再次点击录制按钮停止录制
6. 打开文件页，按 `scene/seq` 找到本次录制
7. 手动选择要上传的录制目录并上传

## 5. 开始录制

主页面中央是录制按钮。

点击录制按钮后，软件会先做两件事：

- 检查相机权限
- 检查麦克风权限

如果麦克风权限未开启，系统会提示你：

- 去设置开启权限
- 或者选择“无音频继续”

如果你选择“无音频继续”，本次录制仍然可以进行，只是不会包含音频。

## 6. 录制前设置

在真正开始录制前，软件会弹出“录制前设置”面板。该面板用于确定本次录制属于哪个场景、哪个序列，以及是否参与双目分组。

### 6.1 场景类型

你需要选择本次录制的场景类型：

- 纯场景
- 带人

这会影响后续服务端如何区分数据类型。

### 6.2 Scene 名称

`Scene` 表示一个场景级别的归档单位，通常对应：

- 一个地点
- 一套布置
- 一次场景配置

如果你没有手动填写，软件通常会给出一个默认建议值，例如：

```text
scene_20260414_183835
```

建议：

- 同一地点、同一布置下连续录制的数据，尽量使用相同的 `sceneName`
- 如果场地或布置发生变化，建议新建一个新的 `sceneName`

### 6.3 Seq 名称

`Seq` 表示某个场景下的一次采集序列。

当前版本通常默认使用：

```text
seq0
```

建议：

- 同一次同步采集的多条视频，使用相同的 `seqName`
- 如果是同一场景下新的采集轮次，可以改成新的 `seqName`

### 6.4 双目分组

如果本次录制不参与双目同步采集，可以不填写分组号。

如果本次录制要与另一台设备上的视频组成同一组，则需要手动填写 `Group ID`。

例如：

```text
group_20260414_183835
```

注意：

- 当前版本没有扫码加入组
- 两个设备必须手动输入**完全相同**的 `Group ID`
- 仅 `Group ID` 相同还不够，`sceneName` 和 `seqName` 也必须相同，服务端才会落到同一目录

### 6.5 确认开始录制

填写完成后，点击：

```text
确认并开始录制
```

点击后会发生以下事情：

- 创建本次录制的本地会话目录
- 立即写入本次录制的 `upload_context.json`
- 开始真正录制

如果点击取消，则不会开始录制。

## 7. 录制中

录制开始后：

- 主界面顶部会显示录制时长
- 中央按钮会变成停止态
- 若当前无音频录制，本次会按无音频模式继续

录制过程中请尽量避免：

- 频繁切到后台
- 锁屏
- 强制关闭 App

## 8. 停止录制

再次点击录制按钮即可停止录制。

停止后当前版本的行为是：

- 软件完成停止录制
- 本次数据保存在本地
- 不会再次弹出同一套录制配置面板

也就是说，`scene/seq/group` 信息已经在录制开始前确定好了，停止后只做本地保存。

## 9. 打开本地文件

在主页面右下角点击文件夹按钮，可进入文件浏览页。

当前文件浏览页不是按真实目录名直接平铺，而是按以下结构展示：

```text
scene
  -> seq
    -> recording
```

你可以按这个顺序逐层进入：

1. 先选择一个 `scene`
2. 再选择这个 `scene` 下的 `seq`
3. 再看到该 `seq` 下的录制目录

这样更方便按业务结构管理本地录制结果。

## 10. 上传录制结果

进入某个录制目录后，可以对该目录执行上传。

上传时：

- 软件会读取该录制目录中的 `upload_context.json`
- 将 `sceneName`、`seqName`、`captureType`、`pairGroupId` 一并带给服务器
- 服务端根据这些字段决定归档路径和分组关系

如果某个录制是双目同步采集的一部分，那么为了让两条视频上传到同一个目录，必须保证两端：

- `captureName` 相同
- `sceneName` 相同
- `seqName` 相同
- `pairGroupId` 相同

其中：

- 同目录主要依赖 `captureName + sceneName + seqName`
- 同组关系主要依赖 `pairGroupId`

## 11. 推荐使用方式

### 11.1 单人普通录制

适合普通单机采集：

1. 点击录制
2. 选择场景类型
3. 填写或确认 `sceneName`
4. 保持 `seq0`
5. 不填写 `Group ID`
6. 开始录制
7. 停止后在文件页上传

### 11.2 单人多段连续录制

适合同一地点连续拍多段：

1. 第一段录制时新建 `sceneName`
2. 后续同场景录制沿用相同 `sceneName`
3. 根据需要决定是否沿用同一个 `seqName`

### 11.3 双人同步录制

适合两台设备同时拍摄：

1. 两位拍摄者提前约定同一个：
   - `sceneName`
   - `seqName`
   - `Group ID`
2. 两边分别点击录制
3. 在录制前设置里手动输入完全一致的值
4. 各自开始录制
5. 各自停止录制
6. 各自在文件页上传

这样服务端才能把它们归到同一个 `scene/seq` 目录，并识别为同组数据。

## 12. 常见问题

### 12.1 停止录制后为什么没有再弹配置面板？

因为当前流程已经改成：

- 录制前先确认配置
- 录制后只做本地保存

这可以避免停止后重复填写，也能保证本地目录从创建时就已经绑定到正确的 `scene/seq`。

### 12.2 两个人都填了同一个分组号，为什么服务器目录还是不同？

因为只填同一个 `Group ID` 还不够。

你还必须保证两边的：

- `sceneName`
- `seqName`
- `captureName`

也一致，服务端才会落到同一目录。

### 12.3 没开麦克风权限还能录吗？

可以。

如果你选择“无音频继续”，软件仍会正常录制，只是本次录制不会包含音频。

### 12.4 文件浏览页为什么按 `scene/seq` 分组，而不是直接显示 `recording_xxx`？

因为按业务分组查看更适合后续上传与管理，尤其适合：

- 同一场景多次录制
- 双目同步采集
- 回头筛选同一个 `scene/seq` 下的多段数据

## 13. 建议命名规则

为了减少混乱，建议使用统一命名方式：

- `sceneName`

```text
scene_YYYYMMDD_HHMMSS
```

- `seqName`

```text
seq0
seq1
seq2
```

- `Group ID`

```text
group_YYYYMMDD_HHMMSS
```

## 14. 当前版本限制

- 不支持扫码建组
- 不支持扫码加入同步组
- 双人同步采集需要手动约定并输入相同的 `scene/seq/group`
- 停止录制后不会自动上传，需要在文件页手动选择上传

## 15. 总结

当前版本的推荐使用方式是：

- 在录制前先确定 `scene/seq/group`
- 录制结束后只保存在本地
- 再从文件浏览页按 `scene/seq` 找到对应录制并上传

如果是双人同步采集，请务必保证双方手动填写一致的：

- `sceneName`
- `seqName`
- `Group ID`

这样服务器才能正确识别它们属于同一组，并落到同一个目录中。
